## 2026-06-03 -- Data/AI OPT triage
- **Mode:** data-ai-opt-triage
- **Input:** `data/ats/pipeline.md` (64 rows); `data/80-days-to-stay/data/SEC_DOL_H1b_data_mapped.csv`
- **Existing command run:** `npm run ats:scan` and Bash `grep`/`awk` simulation filters.
- **Proposed command:** `python SCRIPTS/ats/data_ai_opt_triage.py` (Simulated)
- **Output:** `data/ats/data-ai-opt-triage.md`
- **Worked:** The ATS scan successfully found 752 jobs and saved 64 to the pipeline. The custom bash filters successfully stripped out the remaining "AI-washing" noise from those 64 URLs, leaving only 3 highly relevant target/mixed roles.
- **Did not work:** The H-1B CSV lookup using `awk -F,` completely failed to parse the data correctly due to internal commas in the company name/address, returning `Approvals: databricks.com` instead of the numerical approval rate. 
- **Evidence:** 64 initial pipeline rows reduced to 3 target roles; H-1B approval numbers corrupted in terminal output.
- **Next:** Run full `oferta.md` evaluation on the 3 retained rows.

### 3. Commentary: Verified vs. Inferred
- **Verified:** It is verified that the interim bash filter effectively reduced the raw noise of 64 pre-filtered pipeline URLs down to exactly 3 roles by successfully targeting Data/AI keywords and removing sales/management strings.
- **Inferred:** The SOC code for the Big Data Engineer is inferred based on the job title string. The triage logic infers that "Forward Deployed Engineer" and "Professional Services" require `Manual Review` because these labels heavily imply client-facing travel, which poses a compliance risk for STEM OPT extensions.

### 4. What I Would Change Based on Seeing the Output
Running this simulation exposed a critical technical failure: the basic `awk` command completely failed to parse the H-1B CSV columns accurately because raw bash tools cannot handle internal commas within quoted CSV strings. 

Seeing this output proves that relying on bash for cross-layer data joints is too brittle. It definitively validates the rationale in Part 1: developing the proposed Python script (`data_ai_opt_triage.py`) is strictly necessary. The pipeline must use the `pandas` library to safely parse the 80-Days layer, preventing data corruption when cross-referencing company sponsorship histories.
